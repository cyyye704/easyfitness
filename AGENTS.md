# AGENTS.md

## 项目说明
这是一个名为 EasyFitness 的网页端小程序，使用 Vite + React + TypeScript 构建。

## 项目目标
构建一个饮食与训练记录工具。

核心功能：
- 记录每日体重、饮食、训练和睡眠。
- 支持手动饮食记录，以及由 DeepSeek 整理自然语言生成的待确认草稿。
- 自动汇总每日总热量和总蛋白质。
- 根据当天记录给出评价：加分 / 持平 / 扣分。
- 当前阶段数据只保存在 localStorage。
- AI 解析允许使用最小化 Vercel Serverless Function；暂时不做登录和数据库。

## 常用命令
- 启动开发服务器：npm run dev
- 构建项目：npm run build
- 代码检查：npm run lint

## 工程规则
- 使用 TypeScript。
- 组件保持清晰、可读，不要写得过大。
- 优先使用简单的 React hooks 管理状态。
- 除已批准的 AI Serverless Function 外，不要擅自扩展后端、数据库、用户登录、鉴权或付费服务。
- 不要添加不必要的第三方依赖。
- 页面优先适配手机端。
- UI 保持黑红视觉系统，不使用绿色；风格简洁、理性、克制、有工程感。
- 代码要保证可维护性，不要为了快速实现而写成一坨。

## 工作流程
- Git 工作区中的未提交和未跟踪内容默认视为用户的有效工作，必须保护，不擅自还原、覆盖或清理。
- 在进行较大规模代码修改前，先说明实现计划。
- 修改完成后，总结改了哪些文件。
- 每次实现主要功能后，运行 npm run build。
- 如果构建报错，先修复所有构建错误。
- 不要在没有说明的情况下删除已有功能。
- 不要覆盖用户手写内容，除非明确说明原因。

## 当前 MVP 范围
第一版只需要完成：
1. 今日记录表单。
2. 食物记录新增和删除。
3. 每日热量、蛋白质自动汇总。
4. 训练和睡眠简单记录。
5. 历史记录列表。
6. localStorage 本地保存。
7. 简洁移动端页面。
8. 减脂通识知识模块
9. 自然语言 AI 记录：生成、编辑、确认后追加到现有记录。

## AI 服务长期约束
1. 允许为 AI 功能增加最小化服务端或 Serverless API 层，当前部署平台优先使用 Vercel。
2. 当前允许调用 DeepSeek API。
3. DeepSeek API Key 和其他第三方密钥只能存在于服务端环境变量。
4. 禁止将密钥放入 `VITE_` 前缀变量、前端源码、构建产物、日志或 localStorage。
5. AI 输出只能生成待用户确认的临时草稿，不得未经用户确认直接修改或保存正式记录。
6. 正式记录仍保存在浏览器 localStorage。
7. 服务端当前只负责输入校验、DeepSeek 请求、输出校验、错误处理和基础滥用防护，不负责正式记录持久化。
8. 当前仍不增加数据库、登录注册、鉴权系统、多用户系统或云同步。
9. 保持现有 `DailyRecord → updateRecord → storage.ts → localStorage` 链路。
10. 不因 AI 功能进行无关的全项目状态架构重构。
11. 线上服务不能依赖开发者个人电脑持续运行。
12. 除非用户明确要求，不自动提交、推送或执行生产部署。

## 暂时不要做
- 不做账号登录。
- 不做云端同步。
- 不做数据库。
- 不做复杂图表。
- 不做社交功能。
- 不做未经用户确认的 AI 自动保存、长篇健康分析或多轮对话。
- 不做多用户系统。
